Install

eZ Framework is mainly operated on *nix systems. I used Ubuntu 18.04. I expect eZ Framework will also work on other linux or Unix distros. This project is based on gmp and nettle library which is GPL licensed. Therefore, eZ Framework also follows GPL licence.

eZ 프레임워크는 주로 유닉스 계열에서 돌아갑니다. 저는 우분투 18.04에서 개발하였습니다. 그렇지만, 다른 리눅스나 유닉스에서도 돌아가리라 생각합니다. 이 프로젝트는 GPL 라이선스를 가진 nettle, gmp 라이브러리에 기반합니다. 그러므로, 자동적으로 eZ 프레임워크는 GPL 라이선스를 따릅니다. 

> git clone https://github.com/zetapark/ez
> sudo apt install libmysqlcppconn-dev libjsoncpp-dev \
		nettle-dev libgmp-dev libgmpxx4ldbl
> cd ez
> make

libzeta.so, middle.tls.x and site.src.x will be generated. middle.tls.x is a middle server and site.src.x is a web server. libzeta.so is a dynamic library for our programs.

libzeta.so, middle.tls.x 와 site.src.x 실행파일이 얻어질 것입니다. middle.tls.x는 미들서버이고, site.src.x는 웹 서버입니다. libzeta.so는 동적으로 링크 가능한 라이브러리입니다.

Let's run executables. 
> middle.tls.x &
> site.src.x &

Access https://localhost:4433 with any browser. You will see a very simple form. Let's click button. The value in the text field will change. It is from server side.

Access https://localhost:4433 with a browser. You will see a very simple form. Let's click the button. The value of text field will change. It is from the server side.

https://localhost:4433 을 임의의 브라우저로 접근해 보십시오. 매우 단순한 입력 폼을 볼 수 있을 것입니다. 버튼을 클릭해 봅시다. 텍스트 필드에 있는 값이 변할 것입니다. 이것은 서버측에서 온 것입니다. 

Directory Structure
.
├── database : mysql database utility class
├── obj : object files are genearated here
├── util : log, command line parameter utility
├── tcpip : tcpip, website header
├── tls : middle server and tls
├── src : webserver source
└── www : webserver template

Developers just need to touch two directories that is 'src' and 'www'. 'src' directory is where server side programming sources should be. And web templates are in 'www' directory.

개발자는 단지 두 개의 디렉토리만을 건드리면 됩니다. 그것은 src와 www입니다. src 디렉토리는 서버측 프로그래밍을 위한 소스들이 들어갈 자리입니다. www 디렉토리는 웹 페이지를 위한 템플릿이 들어갈 자리입니다.

루트 디렉토리에 eZ.conf라는 파일이 있다. 이 파일은 서브 도메인에 따라 미들 서버가 연결할 주소를 정의한다.

There is a file named eZ.conf in the root directory. This file defines subdomain to which the middle server will connect. 

					default localhost 2001
					dndd localhost 2001
					ez 192.168.0.14 2001
					www 192.168.0.14 2002

First column defines subdomain name. In case of www.zeta2374.com, it is www. Second column is the ip address in the Local Area Network. Third column is the port number. According to this configuration, the middle server will distribute the connections to appropriate web servers.

첫번째 컬럼은 서브 도메인의 이름이다. www.zeta2374.com 의 경우 www 이다. 두번째 컬럼은 LAN상의 IP 주소이다. 세번째 컬럼은 포트 번호이다. 이 설정 파일에 따라서, 미들 서버는 적절한 웹서버에 연결을 배분할 것이다.
